    - by doing the full recursion each time
    - by caching the values which were already recursively calculated



*** NOTE ***
Absolute values should obviously not be taken seriously, but just as a comparison.



*** DESCRIPTION ***
- Runs a LEARNING session with the specified number of items, USING the cached values when recursing for the activation.
- Takes note of the various values shown in the table.
- Tests how many recursive calls are made during the activation calculation for different number of previous encounters.
- Repeats the above steps, but this time WITHOUT using the cached values and recursing repeatedly.
- NOTE: each LEARNING session consists of 2 STUDY sessions, 30 minutes each, 24 hours apart
